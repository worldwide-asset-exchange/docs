# Signing and User Experience

The signing prompt is where trust is won or lost.

## Product Rules

- explain why the user is signing before the wallet opens
- show which account is active
- keep actions narrow and predictable
- provide a clear outcome after approval or rejection

## Common UX Mistakes

- vague button text before a signature request
- re-prompt loops caused by bad state handling
- silently switching wallets or accounts
- using a production-signing flow without testnet rehearsal
